Direct Kinematics

Configuration

We start with an important concept/definition: configuration.

In the context of fixed-base manipulators, rigid links, and open-chain robots, the configuration of the robot at a given instant in time, \(q\), is nothing more than a column vector with \(n\) elements, where \(n\) is the number of joints, containing the rotation (revolute joint)/translation (linear joint) of each joint at that instant in time.


Direct Kinematics

Configuration

Which direction of rotation/translation is considered positive when the value of \(q_i\) grows, as well as what rotation/translation corresponds to the zero value of the joint (\(q_i=0\)), is a purely conventional choice.

This is common in physics (when setting up a reference frame, you define where zero is and which direction is positive, etc.).

Usually, this comes predefined in the robot, but it can be changed if it is convenient.

See the various configurations on the right, where you can identify the chosen rotation sense (counterclockwise) and zero rotation for both joints.

Direct Kinematics

Configuration

Note that, with the configuration, we can determine the position of any point of the robot.

The general definition of configuration in Mechanics is this: the necessary information to, at that instant in time, determine the position of every particle of a system.

In our case, that information is the rotation/translation of the revolute and linear joints, respectively.
