Robotic Manipulators

Lecture 2: Spatial Description


Spatial Description

A manipulator (or a robot in general) can be viewed as a set of rigid bodies (its links) that move (translate and rotate) as the robot’s motors are actuated.

It is important to be able to describe how each of the links — their position and orientation — change over time. This is especially true for the end-effector.

To do this, we need to know how to describe these bodies in space.

We will start by learning how to describe rotations.



Rotations


Rotations

Rotation Matrix

We have the following definition:

Definition: A rotation matrix is a matrix \(Q \in \mathbb{R}^{3 \times 3}\) with two properties:

  • \(Q^TQ = QQ^T = I_{3 \times 3}\);
  • \(\det(Q) = +1\)

where \(I_{3 \times 3}\) is the \(3 \times 3\) identity matrix.

Rotations

Rotation Matrix

The first condition (\(Q^TQ=I_{3 \times 3}\)) has a geometric interpretation. Suppose it is written as: $$ Q = \LARGE{\Big(}\normalsize{}\begin{array}{ccc} Q_{xx} & Q_{xy} & Q_{xz} \\ Q_{yx} & Q_{yy} & Q_{yz} \\ Q_{zx} & Q_{zy} & Q_{zz} \end{array}\LARGE{\Big)}\normalsize{} = \left(\begin{array}{ccc} Q_{x} & Q_y & Q_z \end{array}\right)$$

where \(Q_x, Q_y,\) and \(Q_z\) are the column vectors that make up the matrix.

Then, reading the first condition in terms of the columns, we deduce that they are orthonormal, i.e., they describe vectors that are orthogonal to each other and of unit length: $$\begin{eqnarray} && Q_x^TQ_y = Q_x^TQ_z = Q_y^TQ_z = 0 \nonumber \\ && Q_x^TQ_x = Q_y^TQ_y = Q_z^TQ_z = 1. \nonumber \end{eqnarray}$$

Rotations

Rotation Matrix

In other words, if we choose a reference frame (a set of three unit vectors orthogonal to each other) \(\mathcal{F}_0\) and consider each column of \(Q\) as a vector written in that frame, once we finish drawing those three vectors corresponding to the three columns, we get another set of orthogonal unit vectors, i.e., another reference frame, \(\mathcal{F}_1\).

Rotations

Rotation Matrix

The second condition, (\(\det(Q)=+1\)), implies that this new frame \(\mathcal{F}_1\) is right-handed according to the right-hand rule.

This means that if you place your right-hand thumb along the \(x\) axis and your index finger along the \(y\) axis, the \(z\) axis is where your palm faces.

Rotations

Rotation Matrix

This procedure (interpreting the columns of a rotation matrix \(Q\) as vectors in a reference frame \(\mathcal{F}_0\), drawing them, and obtaining another frame \(\mathcal{F}_1\)) is very important and will be addressed soon.

We then have the first interpretation of a rotation matrix \(Q\):

A rotation matrix \(Q\) represents a reference frame \(\mathcal{F}_1\), written with respect to another frame \(\mathcal{F}_0\).

Note that the two frames share the same origin!

Rotations

Rotation Matrix

Now we move on to the second interpretation of a rotation matrix \(Q\):

A rotation matrix \(Q\) implements the rotation, around a specific axis (written in reference frame \(\mathcal{F}_0\)) — following the right-hand rule — and of a specific angle, of a point!

Rotations

Rotation Matrix

Example: Consider the rotation matrix \(Q\): $$Q = \Large{\Bigg(}\normalsize{}\begin{array}{ccc} \frac{\sqrt{2}}{2} & -\frac{\sqrt{2}}{2} & 0 \\ \frac{\sqrt{2}}{2} & \frac{\sqrt{2}}{2} & 0 \\ 0 & 0 & 1 \end{array}\Large{\Bigg)}\normalsize{}.$$

Consider a reference frame \(\mathcal{F}_0\), and a point that is written as a column vector \(p\) in \(\mathcal{F}_0\). Then the vector \(p' = Qp\), also written in \(\mathcal{F}_0\), represents point \(p\) rotated by 45 degrees about \(z_0\).

Rotations

Rotation Matrix

So, if: $$p = \Large{\Bigg(}\normalsize{}\begin{array}{c} \frac{\sqrt{2}}{2} \\ \frac{\sqrt{2}}{2} \\ \frac{1}{2} \end{array}\Large{\Bigg)}\normalsize{} \ \ \ , \ \ \ p' = Qp = \Large{\Bigg(}\normalsize{}\begin{array}{c} 0 \\ 1 \\ \frac{1}{2} \end{array}\Large{\Bigg)}.$$

The point \(p'\) is point \(p\) rotated by 45 degrees around \(z_0\).

Rotations

Rotation Matrix

In UAIBot, you can compute the axis and angle (in radians) of a rotation matrix using the function Utils.axis_angle:


import numpy as np
import uaibot as ub

Q = np.array([[0,-1,0],
              [1,0,0],
              [0,0,1]])
axis, angle = ub.Utils.axis_angle(Q)

print(axis)  # Prints [0. 0. 1.]
print(angle) # Prints 1.570 rad

Note that the axis is written in some generic reference frame \(\mathcal{F}_0\). The axis only makes sense once we apply the rotation matrix to a point \(p\) written in a specific reference frame.

Composition of Rotation Matrices

Two successive rotations represented by matrices \(Q_1\) and \(Q_2\) (in that order!) in a reference frame \(\mathcal{F}_0\) form another rotation in \(\mathcal{F}_0\) represented by a rotation matrix \(Q_3=Q_2Q_1\) (the product of the matrices).

Note the order! The second rotation comes first in the product! More recent rotations come on the left!

Also note that the rotation axes are always written in the fixed reference frame \(\mathcal{F}_0\), which does not change throughout the rotations.

Attention! In general, rotations do not commute: $$Q_2Q_1 \not= Q_1Q_2.$$

Conclusion

We can describe the motion of a rigid body if we (1) choose a base reference frame \(\mathcal{F}_0\) and (2) attach a reference frame \(\mathcal{F}_{obj}\) to it.

Then we only need to provide, at every time \(t\), the Homogeneous Transformation Matrix (HTM) that represents the rigid transformation between the fixed reference \(\mathcal{F}_0\) and the object’s frame \(\mathcal{F}_{obj}\) at that time \(t\).

Having how frame \(\mathcal{F}_{obj}\) is at that time lets us say exactly how all the particles of the rigid body are at that time, fulfilling our goal of creating a motion description for the object.
